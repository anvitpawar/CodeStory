Customer Summary Curation Pipeline

Team: Customer Intelligence — Data Engineering  ·  Repo: github.com/customer-intelligence/cust-summary  ·  Generated by AIDE  ·  March 20, 2026

1. Executive Summary

The Customer Summary Curation pipeline ingests data from 9 discrete customer feature stores covering spend behaviour, demographics, risk scores, transaction history, product holdings, digital engagement, complaints, relationship depth, and credit utilisation.

Each feature set is independently summarised by a Tachyon LLM API call using a feature-specific prompt template stored in config/prompt_templates.yaml. The 9 summaries are aggregated into a unified customer profile written to Bigtable (low-latency retrieval) and BigQuery (analytical access). Bankers retrieve the final summary via a Python FastAPI service on Cloud Run keyed on customer_id.
